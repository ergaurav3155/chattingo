Chattingo

A real-time chat application built with React (frontend), Spring Boot (backend), WebSocket and MySQL. This page documents how to run, deploy and maintain the project in a professional production setup.

Project summary

Realtime messaging, JWT auth, scalable containerized services and CI/CD via Jenkins. Designed for deployment on a Linux VPS behind Nginx.

Services
  • Frontend (React) — :3000 / Nginx
  • Backend (Spring Boot) — :8080
  • MySQL — :3306

Overview

Chattingo provides private and group conversations, user profiles, and message persistence. WebSocket (STOMP over SockJS) is used for live chat events while REST endpoints handle account management and history.

Key features

  • Secure sign up / login using JWT
  • Real-time messaging with WebSocket
  • Group chat creation and user presence
  • Dockerized services and Docker Compose orchestration
  • CI/CD pipeline with Jenkinsfile

Architecture

Designed for a simple 3-tier deployment: frontend served by Nginx, backend Spring Boot API, and a MySQL database. WebSocket connections flow directly between frontend and backend (proxied by Nginx in production).

Getting started

  1. Fork & clone the repository: git clone https://github.com/YOUR_USERNAME/chattingo.git
  2. Copy environment templates and update secrets: cp backend/.env.example backend/.env and cp frontend/.env.example frontend/.env
  3. Build and run (recommended): docker-compose up --build -d
  4. Open the frontend at: http://localhost:3000 or your configured domain.

Important commands

docker-compose up --build -d
# Stop and remove containers
docker-compose down -v
# View backend logs
docker-compose logs -f backend

Production deployment notes

  • Use Docker Compose with a separate production compose file (docker-compose.prod.yml) that pins image tags.
  • Use Nginx as a reverse proxy to terminate TLS and proxy traffic to the React app and backend. Example server block included in repo.
  • Use Jenkins to build Docker images, scan for vulnerabilities, push to a registry, and deploy to the VPS by pulling tags and restarting the compose stack.
  • Secure secrets with an environment manager or Docker secrets; avoid committing .env to source control.
